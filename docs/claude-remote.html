Quickstart

The exact setup Claude expects.

The most common mistake is pointing Claude at the wrong base path. Keep the base URL at the origin only.

Correct

ANTHROPIC_BASE_URL=https://your-lemonade-host.example.com

Claude will request https://your-lemonade-host.example.com/v1/messages.

Wrong

ANTHROPIC_BASE_URL=https://your-lemonade-host.example.com/api/v1

Claude would try /api/v1/v1/messages, which is the wrong route.

Shell setup

export ANTHROPIC_BASE_URL="https://your-lemonade-host.example.com"
unset ANTHROPIC_API_KEY
export ANTHROPIC_AUTH_TOKEN="<your-remote-token>"

export ANTHROPIC_DEFAULT_OPUS_MODEL="Qwen3-Coder-Next-GGUF"
export ANTHROPIC_DEFAULT_SONNET_MODEL="Qwen3-Coder-Next-GGUF"
export ANTHROPIC_DEFAULT_HAIKU_MODEL="Qwen3-Coder-Next-GGUF"
export CLAUDE_CODE_SUBAGENT_MODEL="Qwen3-Coder-Next-GGUF"

export CLAUDE_CODE_ATTRIBUTION_HEADER="0"
export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC="1"

claude

PowerShell setup

$env:ANTHROPIC_BASE_URL = "https://your-lemonade-host.example.com"
Remove-Item Env:ANTHROPIC_API_KEY -ErrorAction SilentlyContinue
$env:ANTHROPIC_AUTH_TOKEN = "<your-remote-token>"

$env:ANTHROPIC_DEFAULT_OPUS_MODEL = "Qwen3-Coder-Next-GGUF"
$env:ANTHROPIC_DEFAULT_SONNET_MODEL = "Qwen3-Coder-Next-GGUF"

claude
Verify First

Check the remote endpoint before blaming Claude.

These three probes catch almost every integration problem: version mismatch, wrong model ID, or wrong auth.

1. Server health

curl https://your-lemonade-host.example.com/api/v1/health \
  -H "Authorization: Bearer <your-remote-token>"

Look for "version":"10.0.0" or newer.

2. Model list

curl https://your-lemonade-host.example.com/v1/models \
  -H "Authorization: Bearer <your-remote-token>"

Use the exact id returned here as your Claude model value.

3. Messages route

curl https://your-lemonade-host.example.com/v1/messages \
  -H "Content-Type: application/json" \
  -H "anthropic-version: 2023-06-01" \
  -H "Authorization: Bearer <your-remote-token>" \
  --data '{"model":"Qwen3-Coder-Next-GGUF","max_tokens":8,"messages":[{"role":"user","content":"hi"}]}'

If this returns a message payload, Claude Code can use that host.

When To Use It

Remote Claude is useful when your editor and your accelerator live on different machines.

Laptop + workstation

Keep Claude Code on a thin client while Lemonade runs on a desktop with a larger GPU, more RAM, or an NPU.

Team demo host

Expose one controlled Lemonade endpoint over HTTPS and let multiple developers point Claude at the same tuned model stack.

VPN-only infra

Use a private address when you trust the network, or put Lemonade behind a reverse proxy for a cleaner public URL.

Troubleshooting

Three failure modes show up again and again.

Auth conflict

If both ANTHROPIC_API_KEY and ANTHROPIC_AUTH_TOKEN are set, Claude may choose the wrong one. Unset the API key when using bearer-style Lemonade auth.

Wrong model name

Use the exact model id from /v1/models. A checkpoint name or local filename is not the same thing.

Old Lemonade build

Remote Claude relies on /v1/messages. Upgrade to Lemonade 10.0.0+ if that route returns not_found.
