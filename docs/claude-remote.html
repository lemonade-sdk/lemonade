Quickstart

Set four env vars, then run Claude.

The most common mistake is including a path in the base URL. Set ANTHROPIC_BASE_URL to the origin only — Claude appends /v1/messages on its own.

Correct — origin only

ANTHROPIC_BASE_URL=https://your-lemonade-host.example.com

Claude appends /v1/messages automatically, so the final request hits https://your-lemonade-host.example.com/v1/messages — exactly what Lemonade expects.

Wrong — path included in base URL

ANTHROPIC_BASE_URL=https://your-lemonade-host.example.com/api/v1

Claude still appends /v1/messages, producing /api/v1/v1/messages. That path does not exist and returns 404.

Shell setup

export ANTHROPIC_BASE_URL="https://your-lemonade-host.example.com"
unset ANTHROPIC_API_KEY
export ANTHROPIC_AUTH_TOKEN="<your-remote-token>"

export ANTHROPIC_DEFAULT_OPUS_MODEL="Qwen3-Coder-Next-GGUF"
export ANTHROPIC_DEFAULT_SONNET_MODEL="Qwen3-Coder-Next-GGUF"
export ANTHROPIC_DEFAULT_HAIKU_MODEL="Qwen3-Coder-Next-GGUF"
export CLAUDE_CODE_SUBAGENT_MODEL="Qwen3-Coder-Next-GGUF"

export CLAUDE_CODE_ATTRIBUTION_HEADER="0"
export CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC="1"

claude

PowerShell setup

$env:ANTHROPIC_BASE_URL = "https://your-lemonade-host.example.com"
Remove-Item Env:ANTHROPIC_API_KEY -ErrorAction SilentlyContinue
$env:ANTHROPIC_AUTH_TOKEN = "<your-remote-token>"

$env:ANTHROPIC_DEFAULT_OPUS_MODEL = "Qwen3-Coder-Next-GGUF"
$env:ANTHROPIC_DEFAULT_SONNET_MODEL = "Qwen3-Coder-Next-GGUF"

claude
Verify First

Run three checks before launching Claude.

These probes isolate the three most common failure points in order: server version, model ID, and the messages route itself.

1. Confirm server version

curl https://your-lemonade-host.example.com/api/v1/health \
  -H "Authorization: Bearer <your-remote-token>"

Expected: a JSON response containing "version":"10.0.0" or higher. A lower version means the /v1/messages route is not available — upgrade before continuing.

2. Get the exact model ID

curl https://your-lemonade-host.example.com/v1/models \
  -H "Authorization: Bearer <your-remote-token>"

Expected: a JSON list of objects. Copy the id string exactly — including case — and use it as your Claude model env var. Do not guess or abbreviate.

3. Send a test message

curl https://your-lemonade-host.example.com/v1/messages \
  -H "Content-Type: application/json" \
  -H "anthropic-version: 2023-06-01" \
  -H "Authorization: Bearer <your-remote-token>" \
  --data '{"model":"Qwen3-Coder-Next-GGUF","max_tokens":8,"messages":[{"role":"user","content":"hi"}]}'

Expected: a JSON message payload with a content array. If this succeeds, Claude Code will work with the same host and token.

When To Use It

Use remote Claude when your editor and your GPU are on different machines.

Laptop + workstation

Run Claude Code on a lightweight laptop while Lemonade hosts the model on a nearby desktop with a larger GPU, more VRAM, or a dedicated NPU. You get local editing with remote compute.

Shared team endpoint

Stand up one Lemonade instance behind HTTPS and give every developer on the team the same base URL. Everyone uses the same model version — no per-machine setup required.

Private network deployment

Point Claude at a LAN or VPN address for air-gapped or compliance-sensitive environments. For a public URL, put Lemonade behind a reverse proxy such as nginx or Caddy.

Troubleshooting

The three failure modes you will actually hit.

Auth conflict

What you see: 401 Unauthorized or unexpected prompts to log in to Anthropic.

Cause: ANTHROPIC_API_KEY is still set. When both it and ANTHROPIC_AUTH_TOKEN are present, Claude may send the wrong credential.

Fix: Run unset ANTHROPIC_API_KEY (or Remove-Item Env:ANTHROPIC_API_KEY in PowerShell) before launching Claude.

Wrong model name

What you see: Model not found error or an empty response immediately after sending a message.

Cause: The model env var contains a checkpoint filename, a partial name, or the wrong case — none of which match what Lemonade has loaded.

Fix: Run the model-list check above and copy the id value exactly as returned.

Outdated Lemonade build

What you see: /v1/messages returns 404 or "error":"not_found".

Cause: The Anthropic-compatible messages endpoint was added in Lemonade 10.0.0. Older builds do not expose it.

Fix: Upgrade the server to Lemonade 10.0.0 or later, then re-run the health check to confirm the version.
