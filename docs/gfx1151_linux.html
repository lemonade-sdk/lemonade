Kernel Update Required
Your kernel is missing the Context-Wave-Save-Restore (CWSR) fix for Strix Halo (gfx1151)

What's the problem?

Your system has a Strix Halo GPU (gfx1151), but the running Linux kernel is missing a fix that correctly exports CWSR (Context Wave Save/Restore) properties and handles context switching for this architecture. Without the fix, any GPU compute operation triggers a GCVM_L2_PROTECTION_FAULT in the amdgpu driver, so ROCm backends like llama.cpp-ROCm, vLLM-ROCm, and stable-diffusion.cpp-ROCm hang or crash.

Technical details:

  • Strix Halo optimization guide (AMD)
  • The fix is upstream in kernel 6.18.4+.
  • Some vendor/OEM kernels backport it to 6.17.x, but coverage is inconsistent — verify with the check in step 4 below.

How to fix this

For Ubuntu 24.04 (tested) Recommended

Install a mainline 6.18.4+ kernel

Ubuntu's linux-oem-24.04 ships a 6.17.x kernel that does not include the fix in all builds. The most reliable path is to install a mainline kernel from kernel.ubuntu.com/mainline.

cd /tmp VERSION=6.18.22 BUILD=202604111411 BASE=https://kernel.ubuntu.com/mainline/v${VERSION}/amd64 curl -LO "${BASE}/linux-headers-${VERSION}-061822_${VERSION}-061822.${BUILD}_all.deb" curl -LO "${BASE}/linux-headers-${VERSION}-061822-generic_${VERSION}-061822.${BUILD}_amd64.deb" curl -LO "${BASE}/linux-image-unsigned-${VERSION}-061822-generic_${VERSION}-061822.${BUILD}_amd64.deb" curl -LO "${BASE}/linux-modules-${VERSION}-061822-generic_${VERSION}-061822.${BUILD}_amd64.deb" # Workaround for a run-parts incompatibility in 24.04 when installing mainline kernels: sudo mkdir -p /usr/share/kernel/preinst.d /usr/share/kernel/postinst.d \ /usr/share/kernel/prerm.d /usr/share/kernel/postrm.d sudo dpkg -i linux-*.deb sudo update-initramfs -c -k ${VERSION}-061822-generic sudo update-grub sudo reboot

If you also have amdgpu-dkms installed from an older ROCm release, it may override the kernel's built-in driver with a broken version. See the next box.

If you have amdgpu-dkms installed

Make sure it's from ROCm 31.20 or newer

The DKMS module from the default Radeon repository (amdgpu/30.30) is amdgpu 6.16.13, which also page-faults on gfx1151. Point apt at the 31.20 repository so the DKMS build tracks a working driver:

sudo sed -i 's|amdgpu/30\.30/|amdgpu/31.20/|g' /etc/apt/sources.list.d/amdgpu*.list sudo apt update sudo apt install -y amdgpu-dkms amdgpu-dkms-firmware sudo reboot

If you don't need ROCm's userland libraries (the vLLM backend ships its own), you can skip DKMS entirely and rely on the kernel's built-in amdgpu — that's the simplest configuration.

For other Linux distributions

Upgrade to a kernel with the CWSR fix

Install kernel 6.18.4 or later, or a vendor kernel with the fix backported. Examples:

  • Fedora/RHEL: sudo dnf update kernel
  • Arch Linux: sudo pacman -Syu linux
  • openSUSE: sudo zypper update kernel-default

After updating, reboot and run the verification in step 4 below to confirm the fix is active.

Verify the fix is active

After rebooting, both checks below should succeed.

1

Kernel version

uname -r

Should report 6.18.4 or newer (or a vendor kernel with the fix backported).

2

CWSR properties exported

grep -E "cwsr_size|ctl_stack_size" /sys/class/kfd/kfd/topology/nodes/*/properties

You should see both cwsr_size and ctl_stack_size listed, with non-zero values.

3

GPU compute end-to-end

Sysfs properties alone aren't sufficient — we've seen them appear while the GPU still page-faults on any kernel dispatch. Run a minimal HIP test:

cat <<'EOF' > /tmp/hip_check.cpp #include <hip/hip_runtime.h> #include <stdio.h> __global__ void k(int *d) { *d = 42; } int main() { int *d, h = 0; hipMalloc(&d, sizeof(int)); k<<<1,1>>>(d); hipDeviceSynchronize(); hipMemcpy(&h, d, sizeof(int), hipMemcpyDeviceToHost); hipFree(d); printf("GPU dispatch OK, value=%d\n", h); return h == 42 ? 0 : 1; } EOF hipcc /tmp/hip_check.cpp -o /tmp/hip_check timeout 30 /tmp/hip_check

You should see GPU dispatch OK, value=42. If the test hangs or prints nothing, the kernel/driver combo is still broken — check dmesg | grep amdgpu | tail for page-fault entries.
